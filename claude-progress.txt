## FEATURES ADDED (quality/features.json — 30 → 34 total)

ARCH-001: Webhook event idempotency — webhook_events table with UNIQUE event_id prevents
          duplicate credit grants on Stripe webhook retries.

ARCH-002: Retention cron job — hourly job deletes files from both storage buckets for
          expired packs, secured with CRON_SECRET.

ARCH-003: Credit deduction/restoration via SECURITY DEFINER Postgres functions — deduct_credit(),
          restore_credit(), add_credits() — atomic, RLS-bypassing, never raw UPDATE.

ARCH-004: Signed URL download system — 15-min server-issued signed URLs, ownership validated
          on every download, raw storage paths never exposed.

## KEY ARCHITECTURAL DECISIONS

1. Monolith-first: Single Render service. No microservices at V1 volume. Chromium (+300MB
   memory) is acceptable — simpler than a separate PDF worker service.

2. Triple-layer access: Supabase Auth JWT (who you are) + RLS (table-level rules) +
   explicit server-side ownership check (belt-and-suspenders on every API route).

3. SECURITY DEFINER functions: Credits are never mutated via raw UPDATE from application code.
   All credit operations go through Postgres functions owned by postgres user. This ensures
   atomicity and prevents RLS bypass attacks on user_credits table.

4. Async 202 generation: POST /generate returns immediately with 202. Client polls
   /generate/status every 3 seconds. Prevents connection timeout on slow Puppeteer renders.

5. Idempotent webhooks: INSERT-before-grant pattern using UNIQUE(event_id) on webhook_events.
   No SELECT-then-INSERT race condition. Duplicate webhooks return 200 silently.

6. Signed URLs only: pack-pdfs bucket is server-write-only. No client can read the bucket
   directly. 15-minute signed URLs are the only valid download mechanism.
